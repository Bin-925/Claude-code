# 이슈 트래커: 로컬 마크다운

이 저장소의 이슈와 PRD는 `.scratch/` 아래 마크다운 파일로 관리됩니다.

## 규칙

- 기능 하나당 디렉토리 하나: `.scratch/<기능-슬러그>/`
- PRD 파일: `.scratch/<기능-슬러그>/PRD.md`
- 구현 이슈: `.scratch/<기능-슬러그>/issues/<NN>-<슬러그>.md` (01부터 번호 부여)
- 트리아지 상태는 각 이슈 파일 상단의 `Status:` 줄에 기록 (역할 문자열은 `triage-labels.md` 참고)
- 댓글과 대화 내역은 파일 하단의 `## 댓글` 섹션에 추가

## 스킬이 "이슈 트래커에 게시"라고 할 때

`.scratch/<기능-슬러그>/` 아래 새 파일을 생성합니다 (디렉토리가 없으면 함께 생성).

## 스킬이 "관련 티켓 가져오기"라고 할 때

해당 경로의 파일을 읽습니다. 사용자가 경로나 이슈 번호를 직접 전달합니다.
